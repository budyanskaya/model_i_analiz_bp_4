# Моделирование и анализ бизнес процессов
## Задача 4. 
*Сценарий:* Обработка жалобы клиента
Описание: Клиент отправляет жалобу → регистрация → классификация (простая / сложная) → если сложная: запуск подпроцесса «Расследование» → формирование ответа → отправка клиенту → завершение.

Задание в StormBPMN:
- Создайте процесс Обработка жалобы – [Фамилия].
Смоделируйте:
- Start Event (Message) → Task «Регистрация» → Task «Классификация» → Exclusive Gateway.
- На ветке «Сложная жалоба» добавьте Sub-Process (Collapsed) с названием «Расследование».
- Внутри подпроцесса (разверните) укажите 2–3 задачи: «Сбор данных», «Анализ», «Формирование заключения».

На диаграмме представлен процесс обработки жалобы клиента с участием системы и менеджера. Процесс начинается с получения жалобы клиента, которая регистрируется в системе и отправляется менеджеру по работе с клиентами. Менеджер классифицирует жалобу, определяя, является ли она простой или сложной. Если жалоба простая, процесс переходит непосредственно к формированию ответа. Если жалоба сложная, запускается подпроцесс расследования специалиста, который включает сбор данных по жалобе, анализ данных и формирование заключения, которое затем отправляется менеджеру. После получения заключения специалиста или при простой жалобе менеджер формирует ответ, который отправляется клиенту, и процесс завершается. Диаграмма демонстрирует разделение потоков обработки в зависимости от сложности жалобы и наличие подпроцесса для детального расследования сложных случаев.
